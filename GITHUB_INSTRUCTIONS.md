# Подробная инструкция по публикации проекта MinMotion на GitHub

В этом руководстве детально описан процесс публикации проекта **MinMotion** на GitHub: от локальной подготовки и настройки Git до работы с релизами и публикации готового собранного приложения.

---

## Шаг 1. Установка и базовая настройка Git

Если на вашем компьютере еще не установлен Git, выполните следующие действия:

1. **Скачайте Git:** Перейдите на сайт [git-scm.com](https://git-scm.com/) и скачайте установщик для Windows.
2. **Установите Git:** Запустите установщик, оставляя параметры по умолчанию (рекомендуется).
3. **Проверьте установку:** Откройте командную строку (`cmd` или PowerShell) и введите команду:
   ```bash
   git --version
   ```
4. **Настройте имя и Email:** (Если делаете это впервые) Эти данные будут привязываться к вашим коммитам:
   ```bash
   git config --global user.name "Ваше Имя"
   git config --global user.email "your_email@example.com"
   ```

---

## Шаг 2. Инициализация локального репозитория

Откройте терминал/командную строку непосредственно в корневой папке проекта (`o:\programs\minmotion_native`) и выполните команды:

1. **Инициализируйте Git-репозиторий:**
   ```bash
   git init
   ```
   *Это создаст скрытую папку `.git` в корне вашего проекта.*

2. **Проверьте статус файлов:**
   ```bash
   git status
   ```
   Вы увидите список файлов. Обратите внимание на то, что благодаря созданному файлу `.gitignore`:
   * Временные папки `bin/`, `obj/` и `publish/` скрыты от Git.
   * Тяжелый бинарный файл `ffmpeg.exe` также автоматически игнорируется, чтобы не засорять Git-репозиторий.

3. **Индексируйте все файлы:**
   Добавьте все подготовленные файлы проекта в индекс для первого коммита:
   ```bash
   git add .
   ```

4. **Сделайте первый коммит:**
   Зафиксируйте текущее состояние проекта:
   ```bash
   git commit -m "Initial commit: Core project structure, GitHub-ready assets & docs"
   ```

5. **Переименуйте основную ветку в `main`:**
   Современный стандарт имени главной ветки на GitHub — `main` (вместо устаревшего `master`):
   ```bash
   git branch -M main
   ```

---

## Шаг 3. Создание репозитория на веб-сайте GitHub

1. Авторизуйтесь на сайте [github.com](https://github.com/).
2. В верхнем правом углу нажмите на иконку **`+`** и выберите **`New repository`** (Новый репозиторий).
3. Заполните настройки репозитория:
   * **Repository name (Имя):** Введите имя проекта, например `minmotion-native` или `MinMotion`.
   * **Description (Описание):** Кратко опишите проект (например: *Легковесный аниматор по ключевым кадрам на C# WinForms & WebView2 (GSAP)*).
   * **Public / Private:** Выберите **Public** (Публичный), чтобы любой мог его увидеть, или **Private** (Приватный) для личного использования.
   * **Важно:** В разделе **Initialize this repository with:** **не выбирайте ничего** (оставьте галочки у *Add a README file*, *Add .gitignore* и *Choose a license* **пустыми**). Мы уже создали все эти файлы локально.
4. Нажмите синюю кнопку **`Create repository`** (Создать репозиторий).

---

## Шаг 4. Связывание локального репозитория с GitHub и отправка кода

После создания репозитория GitHub покажет вам страницу с инструкциями. Вам нужна секция **"…or push an existing repository from the command line"**:

1. **Свяжите локальный репозиторий с удаленным репозиторием на GitHub:**
   Скопируйте команду remote add (замените `ВАШ_ЛОГИН` и `ИМЯ_РЕПОЗИТОРИЯ` на ваши реальные значения с сайта):
   ```bash
   git remote add origin https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПОЗИТОРИЯ.git
   ```

2. **Отправьте (залейте) файлы на GitHub:**
   ```bash
   git push -u origin main
   ```
   *При первом запуске Windows может открыть окно авторизации через браузер или запросить Personal Access Token (Токен доступа). Авторизуйтесь под своим аккаунтом GitHub.*

Готово! После завершения отправки обновите страницу репозитория в браузере, и вы увидите весь свой код, красивое оформление из `README.md` и корректную структуру файлов.

## Шаг 5. Публикация готовой сборки (GitHub Releases)

Очень важно дать пользователям возможность скачать готовую программу, не заставляя их компилировать код через Visual Studio или SDK. Для этого на GitHub используется механизм **Releases**:

### 1. Подготовка архива программы
1. Соберите и опубликуйте чистое приложение из командной строки в корне проекта:
   ```bash
   dotnet publish -c Release -o publish
   ```
2. Перейдите в созданную папку `publish/`. В ней содержатся:
   * `MinMotion.exe` (само скомпилированное приложение);
   * Файлы библиотек WebView2 (например, xml);
   * Папка `web/` со всеми исходными веб-файлами (скриптами, стилями, html), которые необходимы для UI приложения.
3. **Рекомендуется:** Скачайте `ffmpeg.exe` и положите его прямо в папку `publish` рядом с `MinMotion.exe` (так как в репозиторий GitHub он не коммитится, пользователям будет намного удобнее получить его "из коробки" в архиве).
4. Выделите все файлы внутри папки `publish` (или саму папку) и запакуйте их в ZIP-архив. Назовите его, например, `MinMotion_v0.1.9_win64.zip`.

### 2. Создание релиза на GitHub
1. Перейдите на страницу вашего репозитория на GitHub.
2. В правой колонке найдите раздел **Releases** и нажмите **Create a new release** (или *Draft a new release*).
3. Заполните параметры:
   * **Choose a tag:** Введите версию проекта, например `v0.1.9` и нажмите *Create new tag*.
   * **Target:** Выберите ветку `main`.
   * **Release title:** Дайте красивое название релизу, например: `MinMotion v0.1.9 (Beta)`.
   * **Description:** Опишите ключевые изменения или оставьте краткое руководство для пользователей (список горячих клавиш, нововведения).
4. **Загрузка архива:** Перетащите подготовленный ранее ZIP-архив (`MinMotion_v0.1.9_win64.zip`) в специальное поле загрузки под описанием релиза (*Attach binaries by dragging and dropping them here*).
5. Нажмите кнопку **Publish release** (Опубликовать релиз).

Теперь у вашего проекта будет официальная страница загрузки с готовым собранным приложением!

## Как обновлять проект в будущем (Памятка разработчика)

Когда вы внесете новые изменения в код и захотите обновить их на GitHub:

1. Откройте терминал в папке проекта.
2. Добавьте измененные файлы в индекс:
   ```bash
   git add .
   ```
3. Сделайте коммит с описанием изменений (помните о правиле инкремента версии):
   ```bash
   git commit -m "Fix: Описание ваших исправлений в версии 0.1.10"
   ```
4. Отправьте изменения на сервер GitHub:
   ```bash
   git push
   ```
